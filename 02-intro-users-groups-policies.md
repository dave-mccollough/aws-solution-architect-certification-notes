# Introduction tu Users, Groups and Policies

- IAM: Users and Groups

  - IAM - Identiy and Access Management

    - Globally scoped
    - Root Account
      - Created by default
      - Shouldn't be used or shared
    - Users
      - People in org
      - Can be organized into groups
      - Can belong to multiple groups or no groups
    - Groups
      - Contains users
      - Cannot not contain other groups

  - IAM: Permissions
    - Users or Groups can be assigned permissions
    - JSON documents called Policies
      - Define permissions assigned to users or groups
    - Apply Principles of Least Privilege
